## 서론
---
[Fundamental Graphics Lab 프로젝트](https://github.com/taeyoung-no/FundamentalGraphicsLab) 진행 중 코딩 에이전트한테 핑퐁 방식으로 가우시안 블러 구현하라고 했더니 2D workgroup을 사용했으며 shared memory는 사용하지 않았습니다. 핑퐁 방식이기 때문에 2D workgroup보다 1D로 하는 게 더 빠를 것 같았습니다. 동시에 shared memory도 사용하여 전역 메모리 접근 횟수를 낮춰서 최적화하려 했습니다. 그런데 1D workgroup과 shared memory를 사용하는 게 실제로 더 빠른지, 빠르다면 얼마나 더 빠른지 직접 확인하고 싶어졌습니다.

## 이론적 배경
---
### 핑퐁 방식에서 1D workgroup이 2D보다 빠른 이유
> A L1 cache line is 128 bytes and maps to a 128 byte aligned segment in device memory. [NVIDIA Nsight Memory Statistics](https://archive.docs.nvidia.com/gameworks/content/developertools/desktop/analysis/report/cudaexperiments/kernellevel/memorystatisticsglobal.htm)

전역 메모리에서 데이터를 캐시 라인 단위로 가져옵니다. 행 방향 블러에서 1D workgroup을 사용하면 2D를 사용하는 것에 비해 cache hit rate가 높습니다.

### shared memory를 사용하는 게 더 빠른 이유
핑퐁 방식에서 workgroup 크기를 $N$, 가우시안 커널 반지름을 $R$이라 하겠습니다. shared memory를 사용하지 않으면 하나의 thread가 전역 메모리에 $1 + 2R$번 접근합니다. 따라서 하나의 workgroup은 $N(1 + 2R)$번 접근합니다. shared memory를 사용하면 하나의 workgroup이 $N + 2R$번 접근합니다. 즉, shared memory를 사용하면 전역 메모리 참조 횟수가 감소해서 더 빠릅니다.

## 실험
---
workgroup이 2D인 경우와 1D인 경우, shared memory를 사용 안 하는 경우와 하는 경우 각각에 대해 블러에 사용한 GPU 시간을 측정했습니다. 실행 시간 편차가 커서 충분히 오래 측정하고 최댓값을 입력했습니다.

- **2D workgroup**: 2.60 ms
- **2D workgroup + shared memory**: 2.15 ms
- **1D workgroup**: 2.50 ms
- **1D workgroup + shared memory**: 2.10 ms

shared memory 사용 여부는 예상대로 유의미한 차이를 보입니다. 한편 workgroup은 1차원일 때 더 빠르긴 하지만 사소합니다. workgroup의 차원이 유의미한 차이를 보이지 않는 이유에 대해 조사했습니다.

> The L1 cache is optimized for 2D spatial locality, so threads of the same warp that read texture or surface addresses that are close together in 2D space will achieve optimal performance. [NVIDIA NSight Compute Profiling Guide](https://archive.docs.nvidia.com/nsight-compute/2024.2/ProfilingGuide/index.html)

즉, 2D workgroup을 사용하면 L1 캐시의 2D 최적화의 영향을 받습니다. 한편 이론적 배경에서 알아본 바와 같이 1D를 사용하면 캐시 라인을 효율적으로 사용합니다. 따라서 2D를 쓰든 1D를 쓰든 어느 정도 최적화가 보장되기에 workgroup의 차원은 성능에 큰 영향을 미치지 않는 것으로 보입니다. 

## 결론
---
2D workgroup을 사용하나 1D workgroup을 사용하나 성능은 비슷하게 나옵니다. 하지만 shared memory는 사용했을 때 유의미하게 성능이 좋아집니다.

GPU에서는 2D 형식의 데이터에 대해 최적화가 되어 있다는 것을 알게 됐습니다.
